# Reporting transparency map

This repository-level map is intended to help editors, reviewers, and readers locate the main elements of the prediction-study design and reproducibility materials. It is not presented as a substitute for any journal-specific reporting checklist.

| Reporting domain | Location |
|---|---|
| Study purpose and intended use | Abstract; Introduction; Methods (calendar-day next-readiness forecasting) |
| Data source and secondary-analysis scope | Methods: Study design and data source; Data availability; Ethics statement |
| Analytic unit, target, and time alignment | Methods: Analytic unit, target, and eligibility; Figure 1 |
| Predictor definitions and forecast-time availability | Methods: Candidate predictors; Figure 1; Table 2 |
| Eligibility and missing-data transparency | Methods; Figure 2; Supplementary Tables S4--S5 |
| Population models, tuning, and sequential personalisation | Methods; Supplementary Table S1; archived analysis code |
| Validation and transportability scope | Methods: Evaluation settings; Table 3; Discussion |
| Performance, calibration, and uncertainty | Methods: Outcomes and statistical analysis; Table 3; Supplementary Tables S2--S3; Supplementary Figure S3 |
| Sensitivity analyses | Results; Supplementary Tables S2 and S6--S7 |
| Reproducibility and data/code access | Data and Code availability statements; README; reproducibility guide; versioned Zenodo archive |
